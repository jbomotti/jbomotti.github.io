SQL(Structured Query Language> is used to access AND manipulate databases! Data is stored in a Relational Database Management System as tables consisting of columns and rows. Use this beautiful cheat sheet to manipulate data with SQL.

Made easier: SQL Quick Reference

Statements Examples
CREATE DATABASE
Create a new database		 CREATE DATABASE sql_forever
CREATE TABLE
Create a new table		 all_day_sql (id INTEGER, user_name VARCHAR, irl_name VARCHAR)
SELECT
Select data from the table		 SELECT id FROM all_day_sql
UPDATE
Update data in a table		 UPDATE all_day_sql
SET irl_name="Kirigaya Kazuto"
WHERE user_name="Kirito"
DELETE
Deletes data from database		 DELETE FROM all_day_sql
WHERE irl_name="Konno Yuuki"
AND alias="Zekken"
INSERT INTO
inserts new data into a database		 INSERT INTO all_day_sql (worlds)
VALUES ('Aincrad', 'ALfheim')
Clauses and Keywords
ORDER BY SELECT id, worlds
FROM all_day_sql
ORDER BY id DESC
WHERE SELECT * FROM all_day_sql
WHERE weapon="Sword"
INNER JOIN SELECT characters.all_day_sql, npcs.sao_data
FROM all_day_sql
INNER JOIN sao_data
ON characters.all_day_sql=npcs.sao_data
Operators
AND/OR
AND returns if both conditions are true, OR if only one condition		 SELECT * FROM all_day_sql
WHERE world='ALfheim"
AND race = "Spriggan"

SELECT * FROM all_day_sql
WHERE world='ALfheim"
AND (race = 'Spriggan' OR race 'Sylph');
IN
Use this if you know the exact value you want to return for at least one of the columns.		 SELECT * FROM sao_data
WHERE abilities IN ('flight', 'night-vision')
LIKE
Search for a pattern BETWEEN Search between an inclusive range		 SELECT * FROM all_day_sql
WHERE username LIKE '%Sakuya'
